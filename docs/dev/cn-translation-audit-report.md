## 五、差距分析

### 5.1 核心差距（影响可玩性）

1. **pre-re/ 完全空白**：作为 EP13.2 Pre-Renewal 服的核心脚本目录，0% 翻译意味着所有 pre-re 专属 NPC 对话、任务、指引全是英文。
2. **jobs/ 几乎空白（3%）**：转职任务是新手最先接触的核心流程，英文对话会严重影响中文玩家体验。
3. **quests/ 严重不足（21.6%）**：88 个任务文件中 69 个未翻译，包括 EP13.2 关键任务链（veins、nameless）。
4. **EP13.2 专属城镇未翻译**：manuk、splendide 是 EP13.2 新增城镇，NPC 全英文。

### 5.2 次要差距（影响完整性）

5. **merchants/（33.3%）**：商人对话影响交易体验。
6. **kafras/（25%）**：卡普拉服务是基础功能。
7. **re/jobs/（3.8%）**：Renewal 转职任务。
8. **re/instances/（16.7%）**：副本 NPC 对话。
9. **map_msg_chn.conf 缺 214 条**：系统消息不完整。

### 5.3 najoast 源的天花板

najoast 源库自身的覆盖率限制了自动翻译的上限：
- jobs: 仅 3%（1/33 文件含中文）
- quests: 仅 33%
- warps: 0%
- events: 27%

**结论：即使完美利用 najoast 源，也无法超过其自身覆盖率。剩余缺口必须通过增量翻译（AI/人工）补齐。**

## 七、增量翻译技术方案

### 7.1 翻译流水线设计

```
1. 抽取待翻译文本
   └─ 从 NPC .txt 文件中提取 mes "...", select("..."), caption "..." 行
   └─ 生成结构化翻译清单（JSON/CSV）

2. AI 翻译
   └─ 批量调用 LLM 翻译英文对话为中文
   └─ 保持 RO 世界观用语一致（参考已翻译文件的术语）
   └─ 注意：NPC 名称、地名使用 RO 中文惯用翻译

3. 人工校对
   └─ 检查专有名词一致性
   └─ 检查语气、角色性格是否合理
   └─ 检查是否有遗漏的变量引用（如 strcharinfo(0)）

4. 回填与编码
   └─ 使用类似 npc_cn_translate.py 的方式回填翻译
   └─ 严格 GBK 编码输出
   └─ 保留原文件结构（坐标、精灵ID、逻辑代码不变）

5. 校验
   └─ 结构校验：对比翻译前后文件的 NPC 块数量、坐标、标签
   └─ 编码校验：确认文件为有效 GBK
   └─ 游戏内验证：启动服务端，实际对话检查显示效果
```

### 7.2 需要新增的工具

| 工具 | 用途 |
|------|------|
| `tools/extract_untranslated.py` | 从 NPC 文件中抽取未翻译的 mes/select 文本，输出翻译清单 |
| `tools/coverage_report.py` | 自动统计各目录中文覆盖率，输出报告 |
| `tools/apply_translation.py` | 将翻译清单回填到 NPC 文件，GBK 输出 |
| `tools/validate_gbk.py` | 批量校验文件 GBK 编码有效性 |
| `tools/terminology_dict.py` | RO 中文术语词典（地名、NPC名、技能名对照） |

### 7.3 术语一致性

需要建立统一的 RO 中文术语表，包括：
- 城镇名（普隆德拉、吉芬、裴扬、拉赫、威鲁斯、马努克、斯普兰蒂等）
- 职业名（剑士、骑士、十字军、牧师、巫师等）
- NPC 常用称呼（冒险者、旅行者等）
- 系统术语（经验值、基础等级、职业等级等）

## 九、风险与注意事项

1. **GBK 编码限制**：部分 Unicode 字符无法用 GBK 表示，需使用 `errors='replace'` 兜底。
2. **NPC 块匹配**：najoast 旧版与 rAthena master 的 NPC 结构差异可能导致匹配失败，需要人工处理。
3. **Pre-Renewal 专属内容**：部分 NPC/任务只在 pre-re 模式下加载，测试时必须以 Pre-Renewal 配置启动。
4. **术语一致性**：不同翻译者可能使用不同的地名/NPC名翻译，需建立术语表并强制统一。
5. **变量引用保护**：翻译时必须保留 `strcharinfo(0)`、`Zeny`、`getarg(0)` 等脚本变量。
6. **客户端配合**：服务端 NPC 中文化只解决对话文本；物品名、技能名、UI 需要客户端中文资源配合。
